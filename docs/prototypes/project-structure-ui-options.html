第 1 步：看懂项目结构

这一步不是让你记住整个仓库，而是先知道当前 issue 会经过哪几类文件。

click/
|-- src/              源码
|   `-- click/
|       `-- formatting.py
|-- tests/            测试
|   `-- test_formatting.py
`-- docs/             文档

教练提示

  • formatting.py 是行为发生点。
  • test_formatting.py 是最小验证点。
  • docs/ 用来确认公开行为是否有文档约束。
